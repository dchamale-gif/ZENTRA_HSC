🧾 Facturación Mejorada + Testing Local

Sistema completo de facturación con descuentos robustos, integración con saldo de paciente e impresión de reportes.

📌 Módulo de Facturación Mejorada

Este módulo incluye:

  • ✓ Búsqueda y selección de pacientes
  • ✓ Agregación de artículos/servicios ilimitados
  • ✓ Descuentos por porcentaje o cantidad fija
  • ✓ Cálculo automático de IVA
  • ✓ Integración automática con saldo del paciente
  • ✓ Impresión de recibos profesionales
  • ✓ Historial de transacciones

Acceder al módulo de facturación:

→ Abrir Facturación Mejorada

⚠️ Nota Importante

Para usar el módulo completo, necesitas:

  1. Backend ejecutándose en http://localhost:3011
  2. Base de datos con las tablas de facturación
  3. Token de autenticación en localStorage
  4. Pacientes registrados en el sistema

🧪 Testing Local - Sin Servidor Remoto

Aquí puedes probar toda la lógica de facturación, descuentos y saldo usando datos simulados localmente en tu navegador.

1️⃣ Crear Factura de Prueba

Crea una factura con dos items y un descuento del 10%.

2️⃣ Obtener Estado de Cuenta

3️⃣ Registrar Pago

Registra un pago de Q200.00 para el paciente PAC-001.

4️⃣ Ver Datos Almacenados

Visualiza todas las facturas y movimientos creados.

🚀 Pasos de Integración

  1. Copiar los archivos JS y HTML al directorio del proyecto
  2. Copiar los archivos del backend al directorio correspondiente
  3. Ejecutar el script SQL en la base de datos local
  4. Agregar las rutas de facturación en backend/server.js
  5. Reiniciar el servidor backend
  6. Acceder a facturacion-mejorada.html desde el navegador

💾 Estructura de Base de Datos

Se crean las siguientes tablas:

pacientes_saldo
├─ id (PK)
├─ paciente_id (FK)
├─ saldo_pendiente
├─ total_deuda
├─ ultima_transaccion
└─ usuario_actualizo

ventas_mejorada
├─ id (PK)
├─ numero_factura
├─ paciente_id (FK)
├─ items: venta_items_mejorada
├─ descuentos: venta_descuentos_mejorada
├─ totales (subtotal, descuentos, impuestos, total)
└─ metodo_pago

movimientos_paciente
├─ id (PK)
├─ paciente_id (FK)
├─ tipo (factura, pago, abono)
├─ monto
├─ saldo_anterior
├─ saldo_nuevo
└─ fecha

📌 Endpoints API

Método Endpoint Descripción

GET /api/billing/estado-cuenta/:id Obtener estado de cuenta
GET /api/saldo-paciente/:id Obtener saldo actual
POST /api/billing/pagos Registrar pago
